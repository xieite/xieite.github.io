xieite::type_id<>

Defined in header <xieite/meta/type_id.hpp>

A type wrapper. This exists because std::type_identity {} isn't guaranteed to be default-constructible.


Definition

 
template<typename T>
struct type_id {
	using type = T;
};


Example

 
static_assert(std::same_as<xieite::type_id<int>::type_id, int>);
