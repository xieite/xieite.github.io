XIEITE_FN(), XIEITE_FN_THIS(), XIEITE_FN_LOCAL(), XIEITE_FN_MUT()

 Defined in header <xieite/pp/fn.hpp>

Allows a shorthand lambda syntax:
 XIEITE_FN(2, $0 + $1)
which expands to something like:
 []<typename...>(auto&& a, auto&& b) noexcept(noexcept(a + b)) -> decltype(auto) requires(requires { a + b; }) { return a + b; }

This collection of macros allows easily creating lambda expressions by automatically generating noexcept specifiers, requires clauses, bodies, template parameters, and function parameters for them.
The body of a shorthand lambda has access to a parameter pack $ containing all arguments, a template parameter pack $$ containing all template arguments, variables $0 through $n which correspond to individual arguments, and types $$0 through $$n which correspond to individual template arguments, where n is an argument passed to the macro.
Accessing a variable or type which does not correspond to an argument passed to the lambda (when the number of arguments is not greater than the variable's index or the number of template arguments is not greater than the type's index) results in some ambiguously unusable value.


Definitions

 #fn 
#define XIEITE_FN(n_, ...) /* ([](???) static ??? { ??? }) */
Expands to a lambda expression with no capture-default, a static specifier, n_ number of template parameter aliases, and n_ number of function parameter aliases.

#fn-this 
#define XIEITE_FN_THIS(n_, ...) /* ([](this auto, ???) ??? { ??? }) */
Expands to a lambda expression with no captures, no specifiers, an explicit this parameter through variable $this of type $$this, n_ number of template parameter aliases, and n_ number of function parameter aliases.

#fn-local 
#define XIEITE_FN_LOCAL(captures_, n_, ...) /* ([???](this auto, ???) ??? { ??? }) */
Expands to a lambda expression with user-provided captures, no specifiers, an explicit this parameter through variable $this of type $$this, n_ number of template parameter aliases, and n_ number of function parameter aliases. captures_ must be wrapped in parentheses if more than one capture item is passed, otherwise parentheses are optional.

#fn-mut 
#define XIEITE_FN_MUT(captures_, n_, ...) /* ([???](this auto, ???) mutable ??? { ??? }) */
Expands to a lambda expression with user-provided captures, a mutable specifier, an explicit this parameter through variable $this of type $$this, n_ number of template parameter aliases, and n_ number of function parameter aliases. captures_ must be wrapped in parentheses if more than one capture item is passed, otherwise parentheses are optional.


Examples

 #0 
int main() {
	auto add_two = XIEITE_FN(2, $0 + $1);
	auto add_all = XIEITE_FN(, (... + $));
	auto cast = XIEITE_FN(1, static_cast<$$0>($0));

	xieite::dump(add_two(3, 4));
	xieite::dump(add_all(1, 2, 3, 4, 5));
	xieite::dump(cast.operator()<int>(6.28318));
}
